<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You can do it with <code>sed</code> and <code>awk</code>:</p>

<pre><code>$ sed 's/[^"]//g' dat | awk '{ print length }'
2
0
</code></pre>

<p>Where <code>dat</code> is your example text, sed deletes (for each line) all non-<code>"</code> characters and <code>awk</code> prints for each line its size (i.e. <code>length</code> is equivalent to <code>length($0)</code>, where <code>$0</code> denotes the current line).</p>

<p>For another character you just have to change the sed expression. For example for <code>(</code> to:</p>

<pre><code>'s/[^(]//g'
</code></pre>

<p><strong>Update:</strong> <code>sed</code> is kind of overkill for the task - <code>tr</code> is sufficient. An equivalent solution with <code>tr</code> is:</p>

<pre><code>$ tr -d -c '"\n' &lt; dat | awk '{ print length; }'
</code></pre>

<p>Meaning that <code>tr</code> deletes all characters which are not (<code>-c</code> means complement) in the character set <code>"\n</code>.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I would just use awk</p>

<pre><code>awk -F\" '{print NF-1}' &lt;fileName&gt;
</code></pre>

<p>Here we set the field separator (with the -F flag) to be the character <code>"</code> then all we do is print number of fields <code>NF</code> - 1. The number of occurrences of the target character will be one less than the number of separated fields. </p>

<p>For funny characters that are interpreted by the shell you just need to make sure you escape them otherwise the command line will try and interpret them. So for both <code>"</code> and <code>)</code> you need to escape the field separator (with <code>\</code>).</p>

<p>Using <code>tr</code> ard <code>wc</code>:</p>



<pre class="lang-bash prettyprint-override"><code>function countchar()
{
    while IFS= read -r i; do printf "%s" "$i" | tr -dc "$1" | wc -m; done
}
</code></pre>

<p>Usage:</p>

<pre class="lang-bash prettyprint-override"><code>$ countchar '"' &lt;file.txt  #returns one count per line of file.txt
1
3
0

$ countchar ')'           #will count parenthesis from stdin
$ countchar '0123456789'  #will count numbers from stdin
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The answers using <code>awk</code> fail if the number of matches is too large (which happens to be my situation). For the answer from <a href="https://unix.stackexchange.com/users/1165/loki-astari">loki-astari</a>, the following error is reported:</p>

<pre><code>awk -F" '{print NF-1}' foo.txt 
awk: program limit exceeded: maximum number of fields size=32767
    FILENAME="foo.txt" FNR=1 NR=1
</code></pre>

<p>For the answer from <a href="https://unix.stackexchange.com/users/6539/enzotib">enzotib</a> (and the equivalent from <a href="https://unix.stackexchange.com/users/10412/manatwork">manatwork</a>), a segmentation fault occurs:</p>

<pre><code>awk '{ gsub("[^\"]", ""); print length }' foo.txt
Segmentation fault
</code></pre>

<p>The <code>sed</code> solution by <a href="https://unix.stackexchange.com/users/1131/maxschlepzig">maxschlepzig</a> works correctly, but is slow (timings below). </p>

<p>Some solutions not yet suggested here. First, using <code>grep</code>:</p>

<pre><code>grep -o \" foo.txt | wc -w
</code></pre>

<p>And using <code>perl</code>:</p>

<pre><code>perl -ne '$x+=s/\"//g; END {print "$x\n"}' foo.txt
</code></pre>

<p>Here are some timings for a few of the solutions (ordered slowest to fastest); I limited things to one-liners here. 'foo.txt' is a file with one line and one long string which contains 84922 matches.</p>

<pre><code>## sed solution by [maxschlepzig]
$ time sed 's/[^"]//g' foo.txt | awk '{ print length }'
84922
real    0m1.207s
user    0m1.192s
sys     0m0.008s

## using grep
$ time grep -o \" foo.txt | wc -w
84922
real    0m0.109s
user    0m0.100s
sys     0m0.012s

## using perl
$ time perl -ne '$x+=s/\"//g; END {print "$x\n"}' foo.txt
84922
real    0m0.034s
user    0m0.028s
sys     0m0.004s

## the winner: updated tr solution by [maxschlepzig]
$ time tr -d -c '\"\n' &lt; foo.txt |  awk '{ print length }'
84922
real    0m0.016s
user    0m0.012s
sys     0m0.004s
</code></pre>

<p>Yet another implementation that does not rely on external programs, in <code>bash</code>, <code>zsh</code>, <code>yash</code> and some implementations/versions of <code>ksh</code>:</p>

<pre class="lang-bash prettyprint-override"><code>while IFS= read -r line; do 
  line="${line//[!\"]/}"
  echo "${#line}"
done &lt;input-file
</code></pre>

<p>Use <code>line="${line//[!(]}"</code>for counting <code>(</code>.</p>

<p>Another possible implementation with awk and gsub:</p>

<pre><code>awk '{ gsub("[^\"]", ""); print length }' input-file
</code></pre>

<p>The function <code>gsub</code> is the equivalent of sed's <code>'s///g'</code> .</p>

<p>Use <code>gsub("[^(]", "")</code>for counting <code>(</code>.</p>

<p>I decided to write a C program cause I was bored.</p>

<p>You should probably add input validation, but other than that's all set.</p>

<pre class="lang-c prettyprint-override"><code>#include &lt;stdio.h&gt;
#include &lt;string.h&gt;

int main(int argc, char *argv[])
{
        char c = argv[1][0];
        char * line = NULL;
        size_t len = 0;
        while (getline(&amp;line, &amp;len, stdin) != -1)
        {
                int count = 0;
                char * s = line;
                while (*s) if(*s++ == c) count++;
                printf("%d\n",count);
        }
        if(line) free(line);
}
</code></pre>

    <div class="post-text" itemprop="text">
<p>Here is another C solution that only needs STD C and less memory:</p>

<pre class="lang-c prettyprint-override"><code>#include &lt;stdio.h&gt;

int main(int argc, char **argv)
{
  if (argc &lt; 2 || !*argv[1]) {
    puts("Argument missing.");
    return 1;
  }
  char c = *argv[1], x = 0;
  size_t count = 0;
  while ((x = getc(stdin)) != EOF)
    if (x == '\n') {
      printf("%zd\n", count);
      count = 0;
    } else if (x == c)
      ++count;
  return 0;
}
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>We can use <code>grep</code> with <code>regex</code> to make it more simple and powerful. </p>

<p>To count specific character.</p>

<pre><code>$ grep -o '"' file.txt|wc -l
</code></pre>

<p>To count special characters including whitespace characters.</p>

<pre><code>$ grep -Po '[\W_]' file.txt|wc -l
</code></pre>

<p>Here we are selecting any character with <code>[\S\s]</code> and with <code>-o</code> option we make <code>grep</code> to print each match (which is, each character) in separate line. And then use <code>wc -l</code> to count each line.</p>

<p>Maybe a more straight forward, purely awk answer would be to use split.
Split takes a string and turns it into an array, the return value is the number of array items generated + 1.</p>

<p>The following code will print out the number of times " appears on each line.</p>

<pre><code>awk ' {print (split($0,a,"\"")-1) }' file_to_parse
</code></pre>

<pre class="lang-py prettyprint-override"><code>#!/usr/bin/env python2
with open('file.txt') as f:
    for line in f:
        print line.count('"')
</code></pre>

<p>For a pure bash solution (however, it's bash-specific): If <code>$x</code> is the variable containing your string:</p>

<pre><code>x2="${x//[^\"]/}"
echo ${#x2}
</code></pre>

<p>The <code>${x//</code> thing removes all chars except <code>"</code>, <code>${#x2}</code> calculates the length of this rest.</p>

<p>(Original suggestion using <code>expr</code> which has problems, see comments: )</p>

<pre><code>expr length "${x//[^\"]/}"
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<pre><code>grep -n -o \" file | sort -n | uniq -c | cut -d : -f 1
</code></pre>

<p>where grep does all the heavy lifting: reports each character found at each line number. The rest is just to sum the count per line, and format the output.</p>

<p>Remove the <code>-n</code> and get the count for the whole file.</p>

<p>Counting a 1.5Meg text file in under 0.015 secs seems fast.<br>
And does work with characters (not bytes).</p>
